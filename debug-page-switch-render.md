# 页面切换渲染问题调试

## 症状
- 页面切换加载速度慢，动画不够丝滑
- 页面切换时出现重影，上一个页面残留叠在当前页面上
- 部分场景页面内容不正常渲染，需要手动刷新后才完整显示

## 当前范围
- 仅排查前端路由切换、布局容器、过渡动画、侧栏交互
- 暂不修改业务数据逻辑

## 初始假设
1. `AppLayout` 中 `router-view + transition + suspense/scoped style` 的组合导致新旧页面在同一阶段共存或样式未稳定命中，出现重影和错页显示。
2. 页面切换慢并非路由本身慢，而是过渡模式、DOM 重排、布局容器 absolute/relative 叠加导致额外重绘。
3. 偶发空白/需刷新与某些页面组件在切换时未重新挂载或 key/缓存策略不稳定有关。
4. `SideNav` 的可拖拽交互可能与点击导航竞争，造成用户点击后路由状态与视觉内容短暂不一致。
5. `AppLayout` 或页面级滚动/容器样式在切换时改变高度流，导致内容区渲染阶段被遮挡或延迟显现。

## 待收集证据
- 路由切换开始/结束顺序
- 进入页面组件的挂载/卸载顺序
- 是否存在 enter/leave 同时可见
- 点击菜单与实际路由更新是否一致
- 内容区是否出现异常高度/可见性状态

## 证据结论
- 已确认：`SideNav` 点击与路由跳转一致，错页不是由菜单点击错误导致。
- 已确认：以 `route.fullPath` 作为页面 key 时，同页内 query 自动补全会触发额外切换，导致多次进入/离开。
- 已确认：过渡方案让新旧页面在切换窗口期共存，是重影/残留的直接来源。
- 已确认：修复后 `stageChildCount` 从 2/3 降到 1，内容区只保留当前页，重影问题消失。

## 已落地修复
- 移除 `router-view` 上的新旧页面共存式过渡。
- 改为“单页面即时切换 + 容器级轻微过渡”。
- 页面 key 改为仅基于 `route.name + params`，忽略 query，避免同页 query 同步触发重挂载。
- 保留调试埋点用于 post-fix 验证。

## 验证结果
- 用户已确认：`看板 → 收件箱 → AI → 收件箱 → 看板` 路径下，问题已修复。
- post-fix 日志显示：多次切页后 `stageChildCount = 1`，query 变化不再造成额外页面共存。
